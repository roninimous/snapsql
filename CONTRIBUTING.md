# Contributing to SnapsQL

Thank you for your interest in contributing to SnapsQL! We welcome contributions from the community to help make this tool better.

## Code of Conduct

Please be respectful and professional in your interactions. We are here to build cool things together.

## How to Contribute

1.  **Fork the Repository**: Create your own fork of the code.
2.  **Create a Branch**: `git checkout -b feature/amazing-feature`
3.  **Make Changes**: Write clean, documented code.
4.  **Test**: Ensure your changes work as expected.
5.  **Commit**: `git commit -m "Add amazing feature"`
6.  **Push**: `git push origin feature/amazing-feature`
7.  **Pull Request**: Submit a PR to the `main` branch.

## Reporting Bugs

If you find a bug, please open an issue with:
-   Steps to reproduce.
-   Expected behavior.
-   Actual behavior.
-   Screenshots (if applicable).

## Feature Requests

We love new ideas! Open an issue to discuss your feature idea before implementing it to ensure it aligns with the project goals.

## Development Setup

1.  Clone the repo.
2.  `cp .env.example .env`
3.  `composer install`
4.  `npm install && npm run build`
5.  `php artisan key:generate`
6.  `php artisan migrate`
7.  `php artisan serve`
